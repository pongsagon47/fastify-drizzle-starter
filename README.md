## Fastify + Drizzle + TypeScript Starter

Starter template for building REST APIs using **Fastify 5**, **Drizzle ORM**, and **TypeScript** with a **MySQL** database.  
It includes JWT authentication, a complete `users` module, mailer utilities, file uploads, and OpenAPI/Swagger documentation.

---

## Features

- **Fastify 5 + TypeScript**: Typed request/response handling with `fastify-type-provider-zod`.
- **Drizzle ORM + MySQL**: Type-safe queries and migrations.
- **Authentication**:
  - Email/password login.
  - JWT access token using `@fastify/jwt`.
  - Protected routes and role-based access (e.g. `admin`).
- **Users module**:
  - CRUD APIs with pagination and search (`name`, `email`).
  - Avatar upload with image processing (via `sharp`).
- **Mailer & templates**:
  - Nodemailer integration.
  - EJS-based email templates + dev tools to preview/send test emails.
- **Developer experience**:
  - Zod validation + OpenAPI generation.
  - Pretty logging in development with `pino-pretty`.
  - Dev-only routes for testing mail templates.

## Project Structure (High Level)

- **`src/server.ts`**
  - Entry point. Loads environment variables, checks DB connection, builds the Fastify app, starts the HTTP server, and handles graceful shutdown (SIGINT/SIGTERM).
- **`src/app.ts`**
  - Fastify app factory. Registers:
    - Core plugins: CORS, JWT, Swagger, mailer, rate limit, multipart, Zod type provider.
    - Global error handler.
    - Health check route.
    - Feature routes (`auth`, `users`, `dev`).
- **`src/config`**
  - `env.ts`: Zod-based environment validation (reads `.env`).
  - `database.ts`: Exposes Drizzle `db` instance, `checkDatabaseConnection()`, and `connectConfig` (used by Drizzle CLI).
- **`src/db/schema`**
  - `index.ts`: Drizzle schema definitions (e.g. `users`, `posts` tables, `User`, `NewUser`, etc.).
- **`src/db/migrations`**
  - Output folder for SQL migrations generated by `drizzle-kit`.
- **`src/modules/auth`**
  - `auth.route.ts`: HTTP routing for `/api/v1/auth` (login, `GET /me`).
  - `auth.service.ts`: Authentication logic (email/password validation, JWT sign, `me()`).
- **`src/modules/users`**
  - `users.repository.ts`: All DB access for users (find, paginate, create, update, delete, avatar update, relation example with `posts`).
  - `users.schema.ts`: Zod DTOs and query types (e.g. `UserQuery`, `CreateUserDto`, `UpdateUserDto`, `UserResponse`).
  - `users.service.ts`: Business logic (uniqueness checks, hashing, avatar upload, mapping DB model → response).
  - `users.route.ts`: HTTP routing & handlers for `/api/v1/users`.
- **`src/modules/dev`**
  - `dev.route.ts`: Dev-only mailer tools under `/api/v1/dev` (template listing, HTML preview, send test email). Enabled only when `NODE_ENV=development`.
- **`src/shared`**
  - `response.ts`: Helpers for consistent API responses (`successResponse`, `paginatedResponse`, `createMeta`).
  - `errors.ts`: Custom error classes used across modules.
- **`src/middlewares`**
  - `errorHandler.ts`: Centralized error handling for all routes.
  - `authenticate.ts`: JWT auth and role-based authorization.
- **`src/utils`**
  - `time/time.ts`: Time & date helpers (including Thai-friendly formatting).
  - `mailer/*`: Mailer utilities + template renderer.
  - `upload/*`: File upload helpers (e.g. image upload + resize).

Drizzle configuration is defined in `drizzle.config.ts`:

- `schema`: `./src/db/schema/index.ts`
- `out`: `./src/db/migrations`
- `dialect`: `mysql`
- `dbCredentials`: From `connectConfig()` (`src/config/database.ts`)

## Getting Started

### 1. Install dependencies

```bash
pnpm install
```

Installs all runtime dependencies (Fastify, Drizzle, MySQL driver, Zod, etc.) and dev dependencies (TypeScript, tsx, drizzle-kit, pino-pretty).

### 2. Configure environment variables

Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

Then edit `.env` to match your local environment:

```env
NODE_ENV=development
PORT=3800

DB_HOST=localhost
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=your_db_name
DB_SSL_CERT=
DB_SSL_KEY=
DB_SSL_CA=

JWT_SECRET=your-super-secret-key-minimum-32-characters-long
JWT_EXPIRES_IN=7d
```

- **`DB_*`**: Connection info for your MySQL instance.
- **`JWT_SECRET`**: Must be at least 32 characters; used by `@fastify/jwt`.
- **`PORT`**: HTTP server port (default `3800`).

Make sure the database specified by `DB_NAME` exists (create it manually or via your DB admin tool).

## Database & Migrations (Drizzle)

Drizzle is configured via `drizzle.config.ts`:

- Reads DB credentials from `src/config/database.ts` (`connectConfig()`).
- Uses `src/db/schema/index.ts` for schema.
- Writes migrations to `src/db/migrations`.

The following scripts are available in `package.json`:

- **`pnpm db:push`**: Apply the current schema directly to the database (auto-generate SQL and run it). Good for early development.
- **`pnpm db:pull`**: Introspect an existing database into Drizzle schema.
- **`pnpm db:generate`**: Generate SQL migration files from schema changes into `src/db/migrations`.
- **`pnpm db:migrate`**: Apply generated migrations from `src/db/migrations` to the database.
- **`pnpm db:studio`**: Open Drizzle Studio to inspect and manage data.

### Typical migration workflows

#### A. Greenfield development (start from schema)

1. Define or update tables in `src/db/schema/index.ts`.
2. Generate migrations:

   ```bash
   pnpm db:generate
   ```

3. Apply migrations:

   ```bash
   pnpm db:migrate
   ```

4. (Optional) Inspect DB using Drizzle Studio:

   ```bash
   pnpm db:studio
   ```

#### B. Rapid local iteration (schema → DB directly)

During early development, you can skip manual migration files and push schema directly:

```bash
pnpm db:push
```

This synchronizes the current Drizzle schema with your database without going through the migration folder.

#### C. Existing database (DB → schema)

If you already have a MySQL database:

```bash
pnpm db:pull
```

This introspects the existing DB and updates your Drizzle schema.

## Running the Server

### Development

```bash
pnpm dev
```

- Uses `tsx` to run `src/server.ts` in watch mode.
- Server listens on `PORT` from `.env` (default `3800`).
- Pretty logging is enabled in non-production using `pino-pretty`.

Once running:

- API base: `http://localhost:3800`
- Swagger docs: `http://localhost:3800/docs`
- Health check: `http://localhost:3800/health`

### Production build

```bash
pnpm build
pnpm start
```

- `pnpm build`: Compiles TypeScript (`tsc`) to `dist/`.
- `pnpm start`: Runs `node dist/server.js`.

Make sure `NODE_ENV=production` and your `.env` is set correctly in the production environment.

## API Overview

### Auth (`/api/v1/auth`)

- **POST `/login`**
  - Body: `email`, `password` (validated by `loginZod`).
  - Returns: `token` (JWT) + basic user info.
  - Rate limited: max 5 requests per minute per IP.
- **GET `/me`**
  - Protected by JWT (`authenticate` middleware).
  - Returns info of the currently authenticated user.

### Users (`/api/v1/users`)

- **GET `/`**
  - Admin-only (`requireRole('admin')`).
  - Query: pagination (`page`, `limit`) and `search`.
  - Returns: paginated users with meta (total, page, pageSize).
- **GET `/:id`**
  - Authenticated users only.
  - Returns a single user.
- **POST `/`**
  - Public create (adjust easily if you want admin-only).
  - Body: `createUserZod` (name, email, password, etc.).
  - Handles:
    - Email uniqueness check.
    - Password hashing before save.
- **PATCH `/:id`**
  - Authenticated users only (authorization logic handled in service/middleware).
  - Body: `updateUserZod`.
- **PATCH `/:id/avatar`**
  - Authenticated users only.
  - Multipart upload (uses `@fastify/multipart` + `sharp`).
  - Old avatar is deleted; new path saved to DB.
- **DELETE `/:id`**
  - Admin-only (`requireRole('admin')`).

### Dev / Email Tools (`/api/v1/dev`) — development only

Enabled only when `NODE_ENV=development`.

- **GET `/mail/templates`**
  - List built-in email templates and usage hint.
- **GET `/mail/preview/:template`**
  - Preview an email template as HTML in the browser.
  - Query params are injected into the template (with defaults).
- **POST `/mail/send/:template`**
  - Send a test email using the specified template to a given address.

## Swagger & API Docs

Swagger/OpenAPI is configured in `src/app.ts` using:

- `@fastify/swagger` and `@fastify/swagger-ui`

Once the server is running, open:

- `http://localhost:<PORT>/docs`

to explore and test endpoints interactively.

## Extending the Starter

- Add new tables/entities by extending `src/db/schema/index.ts`.
- Create new modules under `src/modules/<feature>` with:
  - `*.schema.ts` for Zod DTOs.
  - `*.repository.ts` for DB access.
  - `*.service.ts` for business logic.
  - `*.route.ts` for Fastify routes.
- Reuse shared helpers (`response.ts`, `errors.ts`, `time.ts`, uploader, mailer) to keep modules consistent.

This starter is intentionally minimal but opinionated, so you can quickly build new APIs while keeping a clean and maintainable structure.
