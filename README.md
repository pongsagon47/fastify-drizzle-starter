## Fastify + Drizzle + TypeScript Starter

This project is a starter template for building REST APIs using **Fastify**, **Drizzle ORM**, and **TypeScript** with a MySQL database. It includes JWT authentication, a basic `users` module, and OpenAPI/Swagger documentation.

---

## Project Structure

- **`src/server.ts`**: Entry point. Loads environment variables, builds the Fastify app, starts the HTTP server, and handles graceful shutdown (SIGINT/SIGTERM).
- **`src/app.ts`**: Fastify app factory. Registers core plugins (CORS, JWT, Swagger, Zod type provider), global error handler, health check route, and feature routes.
- **`src/config`** (not fully listed here):
  - `env.ts`: Zod-based environment validation (reads `.env`).
  - `database.ts`: Exposes Drizzle `db` instance and `connectConfig` used by Drizzle CLI.
- **`src/db/schema`**:
  - `index.ts`: Drizzle schema definitions (e.g. `users` table, types like `User`, `NewUser`).
- **`src/db/migrations`**:
  - Output folder for SQL migrations generated by `drizzle-kit`.
- **`src/modules/users`**:
  - `users.repository.ts`: Encapsulates all database access for users (find, paginate, create, update, delete).
  - `users.schema.ts`: Zod DTOs and query types (e.g. `UserQuery`, `UpdateUserDto`).
  - `users.route.ts` / `users.controller.ts`: HTTP routing & handlers for `/api/v1/users`.
- **`src/modules/auth`**:
  - `auth.route.ts` / `auth.controller.ts`: Login/register, returns JWT tokens, uses `@fastify/jwt`.
- **`src/middlewares/errorHandler.ts`**:
  - Centralized error handling for all routes.

Drizzle configuration is defined in `drizzle.config.ts`:

- `schema`: `./src/db/schema/index.ts`
- `out`: `./src/db/migrations`
- `dialect`: `mysql`
- `dbCredentials`: From `connectConfig()` (`src/config/database.ts`)

## Getting Started

### 1. Clone and install dependencies

```bash
pnpm install
```

This will install all runtime dependencies (Fastify, Drizzle, MySQL driver, Zod, etc.) and dev dependencies (TypeScript, tsx, drizzle-kit, pino-pretty).

### 2. Configure environment variables

Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

Then edit `.env` to match your local environment:

```env
NODE_ENV=development
PORT=3800

DB_HOST=localhost
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=your_db_name
DB_SSL_CERT=
DB_SSL_KEY=
DB_SSL_CA=

JWT_SECRET=your-super-secret-key-minimum-32-characters-long
JWT_EXPIRES_IN=7d
```

- **`DB_*`**: Connection info for your MySQL instance.
- **`JWT_SECRET`**: Must be at least 32 characters; used by `@fastify/jwt`.
- **`PORT`**: HTTP server port (default 3800).

Ensure the database specified by `DB_NAME` exists (create it manually or via your DB admin tool).

## Database & Migrations (Drizzle)

Drizzle is configured via `drizzle.config.ts`:

- Reads DB credentials from `src/config/database.ts` (`connectConfig()`).
- Uses `src/db/schema/index.ts` for schema.
- Writes migrations to `src/db/migrations`.

The following scripts are available in `package.json`:

- **`pnpm db:push`**: Apply the current schema directly to the database (auto-generate SQL and run it). Good for early development.
- **`pnpm db:pull`**: Introspect an existing database into Drizzle schema.
- **`pnpm db:generate`**: Generate SQL migration files from schema changes into `src/db/migrations`.
- **`pnpm db:migrate`**: Apply generated migrations from `src/db/migrations` to the database.
- **`pnpm db:studio`**: Open Drizzle Studio to inspect and manage data.

### Typical migration workflows

#### A. Greenfield development (start from schema)

1. Define or update tables in `src/db/schema/index.ts`.
2. Generate migrations:

   ```bash
   pnpm db:generate
   ```

3. Apply migrations:

   ```bash
   pnpm db:migrate
   ```

4. (Optional) Inspect DB using Drizzle Studio:

   ```bash
   pnpm db:studio
   ```

#### B. Rapid local iteration (schema → DB directly)

During early development, you can skip manual migration files and push schema directly:

```bash
pnpm db:push
```

This synchronizes the current Drizzle schema with your database without going through the migration folder.

#### C. Existing database (DB → schema)

If you already have a MySQL database:

```bash
pnpm db:pull
```

This introspects the existing DB and updates your Drizzle schema.

## Running the Server

### Development

```bash
pnpm dev
```

- Uses `tsx` to run `src/server.ts` in watch mode.
- Server listens on `PORT` from `.env` (default `3800`).
- Pretty logging is enabled in non-production using `pino-pretty`.

Once running:

- API base: `http://localhost:3800`
- Swagger docs: `http://localhost:3800/docs`
- Health check: `http://localhost:3800/health`

### Production build

```bash
pnpm build
pnpm start
```

- `pnpm build`: Compiles TypeScript (`tsc`) to `dist/`.
- `pnpm start`: Runs `node dist/server.js`.

Make sure `NODE_ENV=production` and your `.env` is set correctly in the production environment.

## Users Module Overview

The `users` module demonstrates how to structure feature modules with Drizzle and Fastify.

- **Repository** (`src/modules/users/users.repository.ts`):
  - `findAll(query: UserQuery)`: Paginates users with optional search over `name` and `email`.
  - `findById(id: number)`: Fetches a single user by ID.
  - `findByEmail(email: string)`: Used for authentication (includes password field in full `User` type).
  - `create(data: NewUser)`: Inserts a new user using Drizzle and returns the created row.
  - `update(id: number, data: UpdateUserDto)`: Updates user fields.
  - `delete(id: number)`: Deletes a user by ID.

Routes under `/api/v1/users` (and `/api/v1/auth`) are registered in `src/app.ts` and documented through Swagger.

## Swagger & API Docs

Swagger/OpenAPI is configured in `src/app.ts` using:

- `@fastify/swagger` and `@fastify/swagger-ui`
- `fastify-type-provider-zod` + `jsonSchemaTransform` to turn Zod schemas into OpenAPI schemas
- JWT bearer auth (`bearerAuth`) security scheme

Once the server is running, open:

- `http://localhost:<PORT>/docs`

to explore and test endpoints interactively.
